# KongTrade Collective — Zusammenarbeits-Vision
_Stand: 2026-04-19_

## Grundprinzip

4 Personen (Brrudi, Alex, Tunay, Dietmar) bauen parallel ähnliche Systeme.
Ziel: Voneinander lernen, Ressourcen teilen, autonom bleiben.

**Modell: PEER, nicht Chef.** Brrudi initiiert Infrastruktur,
Entscheidungen sind Konsens.

## Drei Kooperations-Ebenen

### Ebene 1 — Knowledge Sharing (Phase 1, Mai 2026)
- Gemeinsames GitHub-Org "kongtrade-collective"
- Shared KNOWLEDGE_BASE (Bugs, Lessons)
- Shared WALLET_RESEARCH (Whale-Analysen)
- Shared STRATEGY_TEMPLATES (Exit, Filter, Signal-Logik)
- Wöchentlicher Sync-Call (Sonntag Abend, 30 Min)

### Ebene 2 — Resource Sharing (Phase 2, Juni–Juli 2026)
- Geteilter Grok/xAI-API-Account (~$50/Mo / 4 Personen)
- Zentraler Polymarket-Data-API-Proxy
- Shared Nansen/Arkham bei Crypto-Erweiterung
- Gemeinsamer Telegram-News-Bot
- Monatliche Kostenabrechnung

### Ebene 3 — Signal Sharing / Schwarm (Phase 3, Q3 2026)
- Schwarm-Datenbank mit anonymisierten Trade-Outcomes
- Jeder Bot schreibt/liest: Performance pro Wallet,
  pro Markt-Typ, pro Zeitfenster
- Jeder entscheidet autonom — aber mit 4x Datenbasis
- Strategie-Arena: Wöchentlicher Performance-Vergleich

### Ebene 4 — Role Specialization (Phase 4, Q4 2026)
- **Brrudi:** Polymarket + Architektur
- **Alex:** Crypto-Whale-Following (Solana)
- **Tunay:** Sentiment-Bot + News
- **Dietmar:** Backtesting + Analytics
- Jeder teilt seine Domäne als Service

## Governance

### Unverhandelbare Regeln
1. Kein Kapital-Pooling
2. Kein gegenseitiges Handeln erlauben
3. Opt-In Everything — niemand muss alle Features nutzen
4. Kosten transparent abrechnen
5. Bugs müssen gemeldet werden

### Entscheidungslogik
- Kleine Dinge (eigene Konfiguration): Jeder selbst
- Mittlere Dinge (Shared-Service-Änderungen): 3 von 4
- Große Dinge (Commercial Use, B2B): Einstimmig

## Architektur-Vision

- Jeder Teilnehmer hat autonomes Setup (eigener Server, eigene Config)
- Zentrale Shared Services auf EINEM Hetzner (Brrudi's oder neu)
- Jeder Bot liest/schreibt Shared Services per API
- Kein Bot kann andere Bots steuern

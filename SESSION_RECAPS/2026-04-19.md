## Kritische Entscheidungen
**Warum CLOB-Balance statt Allowance?** Das `allowance`-Feld ist immer 0 (künftige ERC20-Deposits). Relevantes Feld: `balance` (aktueller CLOB-Deposit in micro-USDC). Hätte zu False-Negatives bei Allowance-Checks geführt.

**Warum AssetType Enum statt String?** String-Werte gingen ohne gültige assetAddress an API → HTTP 400. Enum erzwingt Korrektheit.

## Lessons
- Doku-Overhead ist *realer Produktivitätskiller*. Automation erst nach manueller Schmerz-Messung (30+ Min → 4h Projektion).
- Monitoring-Checks müssen reale API-Felder lesen, nicht "Logik-Vermutungen".
- Category-Pattern-Matching braucht Test-Sets. 78% falsch kategorisiert = Gap in Validation.
